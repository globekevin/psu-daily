## Image Loading Bug Fix (11:47 AM)

### Root Cause Found
User reported all images not loading. Two bugs:

1. **JS `img.complete` bug**: CSS sets `.card-image img { opacity: 0 }`, JS sets `opacity: 1` on `load` event. If images load before script executes (cache/S3 fast load), `load` never fires → all images stay invisible. Fixed by adding `if (img.complete)` check before adding event listener.

2. **Gannett CDN 406**: Nittany Lions Wire image from `usatoday.com/gcdn/` returns 406 irrespective of Accept/Referer headers. Replaced with SI/Minute Media CDN image (`images2.minutemediacdn.com`, 200 + CORS *).

## Second Image Fix Round (12:00 PM) — Triple-redundancy approach

### Why the first fix didn't show immediately
User took screenshot at 11:55 showing still no images. After debugging, found:
1. CSS line 261 `opacity: 0` was a fragile default — depends on JS to flip it
2. Even with `img.complete` check, the page loads at different speeds; the first paint of a fast-loading image may happen before JS runs
3. **GitHub Pages CDN (Fastly) caches HTML files for several minutes** — even after `git push`, the live page still served the old version for ~25 seconds

### Final Solution (commit d8e3aa1)
- **Inline `style="opacity:1"` on every img tag** — bypasses any CSS cache issues, images are visible from first paint
- **Default CSS `opacity: 1`** instead of 0 — fail-safe: no JS needed
- **Removed `onerror="display:none"`** — broken images leave the container with background gradient (visible problem vs invisible)
- **Use `naturalWidth === 0` check** in JS for more reliable "loaded" detection
- **All 6 image URLs verified 200 OK** via curl

### Lesson
For pages served through GitHub Pages CDN, **inline styles are the most reliable fallback** for critical visual properties. CSS-only fixes may take minutes to propagate.


## GitHub Actions Migration (12:27 PM)

### Decision
User chose GitHub Actions over Tencent Cloud. Key reasons:
- GitHub Actions: free, zero-ops, native GitHub Pages integration
- Tencent Cloud: min ¥50/mo, requires server maintenance
- Neither platform magically solves news sourcing — the core work is writing a search+scrape+AI pipeline

### RSS-only approach rejected
- PSU official RSS: only 26 items, all official press releases
- Most third-party RSS feeds broken (GoPSUSports 404, SI 404, StateCollege 403, Collegian 429)
- Only Onward State (200) + NittanySportsNow (200) work — insufficient for 6 categories

### Final Architecture: Bing Search API + DeepSeek
```
GitHub Actions (cron: UTC 23:00 = Beijing 7:00)
    ↓
build_auto.py:
  1. Bing Search API — search 6 PSU categories (~6 queries/day)
  2. Pick best non-duplicate result per category
  3. Fetch article content (fallback to Bing snippet on error)
  4. Extract og:image from page
  5. DeepSeek API — generate Chinese title + 250-500字 summary
  6. Build index.html, psu-news-YYYY-MM-DD.html, archive.html, archive-catalog.html
  7. Update history.json
  8. git commit + push → GitHub Pages auto-deploy
```
