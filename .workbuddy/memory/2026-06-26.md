### 反爬策略加入自动化任务 (automation prompt 更新)
- **Step 2 重命名为 "Fetch today's news (with anti-anti-scraping)"**，新增完整反爬协议：
  1. **统一浏览器 UA**: Chrome 131 macOS，所有请求统一使用
  2. **双轨抓取策略**: Tier 1 先用 WebFetch；Tier 2 如被拦截则用 curl + 完整 headers（Accept、Accept-Language、Referer: google.com）
  3. **频率控制**: 同域名请求间隔 2-3 秒（`sleep 2`），跨域名可并行
  4. **站点分级策略**:
     - pennlive.com: 激进反爬，始终走 curl fallback，付费墙下取标题+导语+og:image 足够
     - centredaily.com / centredailytime.com: JS 渲染/重定向，curl 返回少时回退 WebSearch
     - collegian.psu.edu / onwardstate.com: 通常无障碍，WebFetch 即可
     - psu.edu/news: 可能限流，2 秒间隔
  5. **兜底机制**: 全部抓取失败则换源或换文章，不空卡片
- **Important rules** 中新增反爬守卫规则
